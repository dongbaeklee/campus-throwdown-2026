# CrossFit Registration Codex Guide

이 프로젝트는 `캠퍼스 스로우다운 2026` 참가 신청 랜딩 페이지와 기획/마케팅/운영/재무/홍보 에이전트 팀 하네스를 포함한다.

## 프로젝트 구조

- `index.html`: 정적 랜딩 페이지
- `agents/team.json`: 에이전트 팀 정의와 대회 사실 정보
- `agents/prompts/`: 에이전트별 프롬프트
- `agents/harness.mjs`: 에이전트 팀 실행 브리프 생성기
- `agents/runs/latest.md`: 가장 최근 실행 결과

## Codex에서 에이전트 팀 사용

사용자가 아래처럼 요청하면 `agents/team.json`과 `agents/prompts/`를 읽고 에이전트 팀을 사용한다.

- "홍보 에이전트 팀 돌려줘"
- "기획, 마케팅, 운영, 재무 에이전트로 실행 브리프 만들어줘"
- "마케팅 리드로 이번 주 모집 전략 짜줘"
- "운영 리드로 신청 흐름 체크리스트 만들어줘"
- "재무 리드로 참가비 기준 손익 점검해줘"
- "기획 리드로 다음 회의 안건 정리해줘"
- "인스타그램 캠페인 만들어줘"
- "제휴 담당자 에이전트로 DM 초안 만들어줘"
- "카피라이터 에이전트로 포스터 문구 뽑아줘"
- "전환 분석가로 신청 페이지 개선안 내줘"
- "운영 검수자로 홍보 문구 확인해줘"

전체 팀 실행이 필요한 경우 다음 명령을 사용한다.

```bash
npm run agents:run -- "<이번 홍보 목표>"
```

실행 후 `agents/runs/latest.md`를 읽고 사용자에게 핵심 결과를 요약한다. 사용자가 원하면 결과를 `index.html`의 문구, FAQ, CTA, 신청 섹션에 반영한다.

## 에이전트 역할

- `planning_lead`: 목표 정의, 핵심 의사결정, 마일스톤, 부서별 요청사항
- `marketing_lead`: 마케팅 목표, 핵심 메시지, 채널 전략, 콘텐츠 요청 브리프
- `operations_lead`: 운영 체크리스트, 리스크 대응, 문의 응대 기준, 검수 필요 항목
- `finance_lead`: 수익 추정, 예산 항목, 손익 리스크, 정산 체크리스트
- `campaign_strategist`: 캠페인 각도, 핵심 메시지, 2주 실행 플랜
- `copywriter`: SNS 캡션, 포스터 헤드라인, CTA, FAQ 문구
- `channel_planner`: 채널별 게시 일정, 소재 포맷, 성과 지표
- `partnership_manager`: 제휴 타깃, DM/메일 초안, 팔로업 문구
- `conversion_analyst`: 전환 리스크, 빠른 개선안, A/B 테스트
- `ops_reviewer`: 사실 정보 검수, 운영 리스크, 게시 전 체크리스트

## 작업 원칙

- 날짜, 장소, 참가비, 모집 인원은 `agents/team.json`의 `hard_facts`를 기준으로 한다.
- 현재 시스템 날짜가 대회 날짜 이후라면 사용자에게 날짜 갱신 필요성을 알려준다.
- 확인되지 않은 표현은 쓰지 않는다. 예: "국내 최대", "공식 인증", 확정되지 않은 스폰서.
- 홍보 문구를 실제 페이지에 반영하기 전에는 운영 검수 관점으로 한 번 확인한다.
- 개인정보 처리방침, 환불, 참가 자격 안내는 실제 운영 문서가 없으면 확정 표현을 피한다.

## 자주 쓰는 Codex 요청 예시

```text
기획, 마케팅, 운영, 재무 에이전트로 이번 주 실행 브리프 만들어줘.
```

```text
홍보 에이전트 팀으로 마감 7일 전 캠페인 만들어줘.
```

```text
카피라이터 에이전트로 인스타 피드 캡션 5개와 스토리 문구 10개 만들어줘.
```

```text
전환 분석가 에이전트로 현재 index.html 신청 섹션 개선안 내고 바로 반영해줘.
```

```text
운영 검수자 에이전트로 latest.md의 홍보 문구 리스크만 체크해줘.
```

```text
제휴 담당자 에이전트로 대학 운동 동아리에 보낼 DM과 이메일 초안 만들어줘.
```
